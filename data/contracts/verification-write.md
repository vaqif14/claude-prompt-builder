NO COMPLETION CLAIMS WITHOUT FRESH VERIFICATION EVIDENCE

Before any status claim:
1. Name the command or observation that proves it.
2. Run or collect it fresh and completely.
3. Read the complete output and exit status.
4. Make the claim with the evidence, or state the real status.
Skipping a step invalidates the claim.

Provable by command means fresh complete output plus exit status, never an assumption.
Read every diff, check side-effects / regressions and silent failures, and validate claimed `file:line` references.
